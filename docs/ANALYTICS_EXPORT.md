# Analytics Export Feature

## Overview

The Analytics Export feature allows executives and claims managers to generate comprehensive PDF and CSV reports of fast-track analytics metrics. Reports include all 5 key performance indicators with professional formatting and date range filtering.

## Features

### Export Formats

1. **PDF Report**
   - Professional formatting with tables
   - Header with metadata (date range, generated by, timestamp)
   - All 5 fast-track metrics with detailed breakdowns
   - Footer with page numbers
   - Ready for stakeholder presentations

2. **CSV Report**
   - Raw data in comma-separated format
   - Same metrics as PDF
   - Suitable for further analysis in Excel/spreadsheets
   - Easy to import into BI tools

### Metrics Included

1. **Fast-Track Eligibility Rate**
   - Total claims processed
   - Fast-track eligible count
   - Eligibility percentage

2. **Auto-Approval Rate**
   - Total fast-track claims
   - Auto-approved count
   - Auto-approval percentage

3. **Processing Time Comparison**
   - Fast-track average (hours)
   - Standard workflow average (hours)
   - Time savings (hours)

4. **Executive Override Frequency**
   - Total fast-track claims
   - Executive override count
   - Override percentage

5. **Risk Distribution**
   - LOW risk count and percentage
   - MEDIUM risk count and percentage
   - HIGH risk count and percentage

## Usage

### Backend (tRPC Procedures)

```typescript
// Export PDF
const pdfResult = await trpc.analytics.exportFastTrackPDF.mutate({
  tenantId: "tenant-001",
  startDate: new Date("2026-01-01"),
  endDate: new Date("2026-12-31"),
});

// Export CSV
const csvResult = await trpc.analytics.exportFastTrackCSV.mutate({
  tenantId: "tenant-001",
  startDate: new Date("2026-01-01"),
  endDate: new Date("2026-12-31"),
});
```

### Frontend (React Component)

```tsx

function ExecutiveDashboard() {

```


## Role-Based Access Control

Only users with the following roles can export analytics reports:

- **Executive** (`role: 'executive'`)
- **Claims Manager** (`role: 'claims_manager'`)

Unauthorized users will receive a `403 FORBIDDEN` error.

## Implementation Details

### Service Layer

**File:** `server/services/analytics/analytics-export.ts`

- `gatherAnalyticsData()` - Collects all 5 metrics in parallel
- `generatePDFReport()` - Creates PDF using jsPDF and jspdf-autotable
- `generateCSVReport()` - Generates CSV string with proper formatting

### API Layer

**File:** `server/routers/analytics.ts`

- `analytics.exportFastTrackPDF` - tRPC mutation for PDF export
- `analytics.exportFastTrackCSV` - tRPC mutation for CSV export

Both procedures:
- Validate user role (Executive/ClaimsManager only)
- Accept tenantId, startDate, endDate as input
- Return filename, data (base64 for PDF, string for CSV), and mimeType

### Frontend Component

**File:** `client/src/components/AnalyticsExportButton.tsx`

Features:
- Date range picker (start and end dates)
- Format selection (PDF or CSV)
- Loading states during generation
- Automatic file download
- Toast notifications for success/error
- Input validation (date range checks)

## Error Handling

### Common Errors

1. **Unauthorized Access (403)**
   ```
   Only executives and claims managers can export analytics reports
   ```

2. **Invalid Date Range**
   ```
   Start date must be before end date
   ```

3. **Missing Data**
   ```
   No claims found for the specified date range
   ```

### Frontend Error Handling

The component displays toast notifications for all errors:

```typescript
toast({
  title: "Export failed",
  description: error.message || "Failed to generate PDF report",
  variant: "destructive",
});
```

## Performance Considerations

- All 5 metrics are fetched in parallel using `Promise.all()`
- Single-query implementations prevent N+1 issues
- PDF generation is synchronous but fast (<1 second for typical reports)
- CSV generation is extremely fast (string concatenation)
- Reports are generated on-demand (no caching)

## Testing

### Manual Testing Steps

1. Log in as Executive or Claims Manager
2. Navigate to analytics dashboard
3. Click "Export Report" button
4. Select date range (e.g., last month)
5. Choose format (PDF or CSV)
6. Click "Export"
7. Verify file downloads automatically
8. Open file and verify all 5 metrics are present
9. Check formatting and data accuracy

### Role-Based Access Testing

1. Log in as Assessor or other non-executive role
2. Attempt to export report
3. Verify 403 error is returned
4. Check error message is displayed in toast
